# システムの目的・ユニーク性（100文字以内）

ポケモンSV初心者が迷わず進めることを目的とした攻略サイト。御三家選択と平均レベルから、挑戦可能なジムを直感的に判断できる点が特徴。

# 要件定義
ポケモンSV 超初心者向けストーリー攻略アプリ

1. システムの目的と背景
■ 背景

ポケモン スカーレット・バイオレットはオープンワールド形式のため、
初心者にとって「次に何をすればよいか」「どこに行けるのか」が分かりにくい。
既存の攻略サイトは情報量が多く、専門用語も多いため、
ポケモン未経験者やライト層が途中で挫折しやすい課題がある。

■ 目的

本アプリは、ポケモンSVを初めて遊ぶ初心者が、
レベルを基準に「今行ける場所」を直感的に判断できることを目的とする。
難しい育成論や専門知識を排除し、「レベルを上げれば進める」
というシンプルな攻略体験を提供する。

2. 使用者（対象ユーザー）
ユーザー層	特徴
ポケモン超初心者	タイプ相性や育成方法が分からない
シリーズ初プレイ	SVが初めての作品
子ども・ライト層	難しい説明が苦手、直感的に理解したい
ストーリー重視層	対戦より物語クリアが目的
3. システム機能（機能要件）
■ 基本機能
No	機能名	概要
1	御三家選択機能	ニャオハ・ホゲータ・クワッスから選択し、以降の表示に反映
2	レベル入力機能	手持ち6匹のレベルを入力し平均レベルを算出
3	ジム・イベント選択	ジム・ヌシ・スター団を一覧から選択可能
4	挑戦可否判定	平均レベルと推奨レベルを比較し挑戦可能か表示
5	初心者向けアドバイス	有利タイプ・レベル不足時の育成方法を表示
6	マップ表示	SV全体マップを画像で表示し位置関係を把握可能

※ 本アプリは「必ず勝てるルート」を提示するのではなく、
初心者が判断に迷わないための目安を提示することを重視する。

4. 技術構成
区分	内容
フロントエンド	Streamlit
バックエンド	Python
データ管理	辞書型データ（ジム・イベント情報）
画像表示	Pillow（PIL）
状態管理	Streamlit Session State
実行環境	Python 3.10以上
ホスティング	Streamlit Community Cloud（想定）

※ 外部APIやデータベースは使用せず、
講師PCでも確実に起動できる構成とする。

5. 非機能要件
区分	要件内容
操作性	ボタン中心のUIで初心者でも迷わない
パフォーマンス	ページ表示は即時（ローカル実行）
可用性	README記載手順のみで起動可能
保守性	ジム・レベル情報はコード内で簡単に編集可能
セキュリティ	認証・個人情報入力なし（閲覧専用）
6. 成果物
区分	内容
成果物	ポケモンSV初心者向け攻略WEBアプリ
主な画面	御三家選択画面、攻略画面
使用技術	Python / Streamlit
納品形態	GitHubリポジトリ + 実行可能アプリ
7. 成功指標（KPI）
指標	目標
初心者が「次に行く場所」を判断できる	YES
レベル不足に気づける	YES
途中で迷わずストーリーを進められる	YES

🧩 総括
本アプリは、複雑化したポケモンSVのストーリー進行を
「レベル」という一つの基準に集約することで、
初心者でも安心して最後まで遊べる攻略体験を提供する。
学習用課題として、実装・操作・起動がシンプルで再現性の高い
初心者ファーストのWEBアプリである。
